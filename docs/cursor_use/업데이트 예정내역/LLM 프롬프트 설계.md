# Life RPG LLM Prompt 설계 및 Ollama 안정화 가이드

이 문서는 **Cursor AI에게 바로 전달하여 코드 개선 작업을 수행할 수
있도록 작성된 명령형 문서**입니다.\
현재 Life RPG 프로젝트에서 **Ollama + Qwen2.5 모델 사용 시 발생하는
문제**를 해결하는 것이 목표입니다.

------------------------------------------------------------------------

# 목표

현재 문제

-   한국어가 아닌 영어 / 일본어 / 중국어가 섞여서 출력됨
-   JSON 출력이 깨짐
-   프롬프트 규칙을 모델이 무시함
-   모델 응답 안정성이 낮음

해결 목표

-   한국어 출력 강제
-   JSON 출력 안정화
-   Ollama 옵션 최적화
-   LLM 응답 검증 로직 추가
-   Life RPG 분석 프롬프트 표준화

------------------------------------------------------------------------

# 1. System Prompt 개선

현재 System Prompt는 너무 약합니다.\
아래 Prompt를 사용하도록 수정하세요.

``` text
당신은 Life RPG 서비스의 분석 AI입니다.

반드시 아래 규칙을 지켜야 합니다.

규칙:
1. 모든 출력은 한국어로만 작성합니다.
2. 영어, 일본어, 중국어 등 다른 언어를 절대 사용하지 않습니다.
3. 외래어도 가능하면 한국어로 표현합니다.
4. 출력에 다른 언어가 포함되면 잘못된 응답입니다.
5. 설명 없이 요청된 형식만 출력합니다.

이 규칙을 반드시 지켜야 합니다.
```

NestJS 코드에서 아래 상수로 정의하세요.

``` ts
const KOREAN_ONLY_SYSTEM = `
당신은 Life RPG 서비스의 분석 AI입니다.

규칙:
1. 모든 출력은 한국어로만 작성합니다.
2. 영어, 일본어, 중국어를 절대 사용하지 않습니다.
3. 요청된 형식 외 텍스트를 출력하지 않습니다.
`;
```

# 2. Ollama 옵션 추가

현재 Ollama 호출에 옵션이 없습니다.

다음 옵션을 추가하세요.

``` ts
options: {
  temperature: 0.2,
  top_p: 0.9,
  repeat_penalty: 1.1,
  num_ctx: 4096
}
```

NestJS 코드 수정 예시

``` ts
const result = await this.post('/api/generate', {
  model: m,
  prompt,
  stream: false,
  options: {
    temperature: 0.2,
    top_p: 0.9,
    repeat_penalty: 1.1,
    num_ctx: 4096
  }
});
```

목적

-   언어 흔들림 감소
-   JSON 출력 안정성 증가
-   응답 품질 개선

# 4. 응답 언어 검증 로직 추가

LLM이 규칙을 무시하는 경우를 대비해 백엔드에서 검증을 수행하세요.

``` ts
function containsNonKorean(text: string) {
  return /[a-zA-Z\u3040-\u30ff\u4e00-\u9faf]/.test(text);
}
```

LLM 응답 검사

``` ts
const response = await this.generate(...);

if (containsNonKorean(response)) {
  this.logger.warn("Non Korean detected. Retrying...");
  return this.generate(...);
}
```

# 5. 활동 분석 프롬프트

``` text
사용자의 활동 로그를 분석하세요.

분석 규칙
- 반드시 한국어로 작성
- 2~3문장으로 작성
- 격려 또는 개선점을 포함

활동 로그:
${activity}

출력 형식:
1줄 요약
2줄 분석
```

예시

오늘은 꾸준히 학습 활동을 진행했습니다.\
특히 단어 학습을 통해 지식 능력이 향상되었습니다.\
내일은 학습 시간을 조금 더 늘리면 더 좋은 성과를 얻을 수 있습니다.

# 8. LLM 비용 절감 구조

LLM 호출을 최소화하기 위해 아래 구조를 유지하세요.

    activity log
       ↓
    rule engine
       ↓
    xp 계산
       ↓
    LLM 분석
       ↓
    daily report

LLM은 다음 작업만 수행하도록 제한합니다.

-   활동 요약
-   일일 분석
-   목표 분석
-   사용자 코칭

XP 계산은 가능한 rule engine으로 처리합니다.
