## 0.0.5

### UX 개선

- 활동 기록 날짜 선택 허용
  - 기존 "오늘만 기록 가능"에서 오늘 포함 최근 7일 이내로 변경
  - 날짜 입력 필드 추가 (미래 날짜 입력 불가)
- 활동 저장 후 XP 피드백 화면 추가
  - 저장 완료 시 획득한 총 XP 및 능력치별 XP 변화 표시
  - Ollama 분석 실패 시 사용자에게 안내 메시지 표시
  - 3초 후 캐릭터 페이지로 자동 이동 (또는 즉시 이동 링크 제공)
- 활동 저장 후 능력치 즉시 갱신
  - 활동 기록 페이지에서 캐릭터 페이지로 돌아올 때 능력치 바 자동 재로드
- 분석 페이지 개선
  - 목표별 "분석 재생성" 버튼 추가
  - 전체 새로고침 버튼 추가
  - 분석 로딩 중 스피너 표시

### 버그 수정

- 활동 캘린더에서 목표가 있는 날에 활동 배지가 표시되지 않던 문제 수정
  - 기존: 목표 없는 날에만 활동 배지 표시, 목표 있는 날에도 중복 조건으로 렌더링 누락
  - 수정: 활동이 있으면 항상 활동 배지 표시

### API 개선

- 활동 생성(`POST /activity`) 응답에 `abilityXpChanges`, `ollamaFailed` 필드 추가
  - `abilityXpChanges`: 능력치별 XP 변화 목록 (`[{ name, xp }]`)
  - `ollamaFailed`: Ollama 분석 실패 여부
- 분석 배치 조회 엔드포인트 추가 (`GET /analysis/goals?characterId=...`)
  - 기존 N+1 방식(목표 N개 × 개별 API 호출)에서 단일 쿼리로 개선
  - 반환: `{ [goalId]: string | null }` 형태의 목표별 최신 분석 내용

### 코드 품질

- `ActivityCreateResultDto`, `GoalAnalysisDto` 타입 정의 추가 (`api.ts`)
- `analysis.getGoalsBatch()` API 클라이언트 메서드 추가
- `CharacterView`에 `refreshKey` prop 추가로 외부에서 능력치 재로드 트리거 가능
- `CreateActivityLogDto` — `content`, `characterId` 필드에 `@IsNotEmpty()` 검증 추가
- `CreateAbilityStatDto` — `xp` 필드에 `@Min(0)` 검증 추가
- CORS 설정을 `localhost:3000`만 허용하도록 강화 (`CORS_ORIGIN` 환경변수로 확장 가능)
